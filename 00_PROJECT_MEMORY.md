# 1. Project Overview

KHAS Lifestyle is a premium Persian lifestyle platform being developed as a fully custom WordPress theme.

This project is NOT intended to become another generic magazine or news website.

The objective is to build one of the highest-quality Persian lifestyle platforms with a focus on:

- simplicity
- trust
- speed
- premium visual design
- long-term scalability
- maintainable architecture

Every technical decision must support these goals.

---

# 2. Project Vision

KHAS Lifestyle should eventually become an ecosystem.

The website is only the first phase.

Future phases may include:

• Premium Membership

• Educational Courses

• Medical Content

• Lifestyle Content

• Digital Products

• AI Services

• Mobile Application

• Community Features

Everything must be developed so these future features can be added without rewriting the project.

# 5. Development Philosophy

This project is developed using AI-assisted engineering.

AI is expected to behave like a Senior Software Engineer, not a code generator.

Every implementation must be:

Readable

Maintainable

Documented

Reusable

Minimal

Production-ready

If there are multiple possible implementations, always choose the simplest architecture that satisfies future scalability.

# 6. Long-Term Product Identity

KHAS Lifestyle is intended to become a trusted lifestyle reference.

It is NOT a clickbait website.

It is NOT a news portal.

It is NOT an SEO-content farm.

It is intended to become a premium digital brand.

---

# 7. Design Philosophy

Visual identity should always feel:

Minimal

Elegant

Modern

Calm

Premium

Professional

Pages should contain generous white space.

Typography should be prioritized over decoration.

Animations should remain subtle.

The interface should never feel crowded.

---

# 8. Performance Philosophy

Performance is a core requirement.

Every page should be designed under the assumption that:

Performance is a feature.

The project should remain lightweight.

Avoid unnecessary JavaScript.

Avoid unnecessary libraries.

Avoid unnecessary plugins.

Only implement what is required.

---

# 9. Code Philosophy

The codebase must always prioritize:

Readability

Consistency

Low complexity

Modularity

Reusability

Avoid duplicated logic.

Avoid giant functions.

Avoid deeply nested code.

Prefer descriptive names.

Document non-obvious decisions.

# 11. Membership System

Membership is a core business feature.

Membership should remain extensible.

# 12. Payment Philosophy

Payment architecture must remain modular.

Business logic should remain separated from presentation.

Future gateways should be replaceable without rewriting large sections of the project.

Never tightly couple payment logic with UI.
